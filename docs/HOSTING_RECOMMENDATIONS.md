| 순위 | 서비스 | 예상 월 비용 | 비고 |
|------|--------|--------------|------|
| **1** | **Fly.io** | **약 $0~3** (사용량 $5 미만이면 청구 면제 정책) | 소형 VM 1대 상시 실행. Docker 배포 필요. |

## 현재 단계 (개발·테스트 중)

| 방식 | 비용 | 비고 |
|------|------|------|
| **Streamlit Community Cloud (현재)** | **무료** | 잠자기 있음 → GitHub Actions 자동 깨우기 사용 중이면 실사용에 가깝게 테스트 가능 |
| **Railway** (테스트용) | **$5/월** 포함 크레딧으로 시작, 소규모면 **월 $5~10** | 상시 실행, 잠자기 없음. 상용화 전 단계에서 가장 추천 |

**정리**: 지금은 **Streamlit Cloud 무료 + 자동 깨우기**로 테스트를 이어가고, 상용화 직전이나 트래픽이 늘면 **Railway**나 아래 옵션으로 이전하는 구도를 추천합니다.

## 상용화 전·초기 단계 추천 호스팅 (우선순위)

Streamlit + Python + Firebase/Gemini 연동을 그대로 쓸 수 있는 곳 기준입니다.

### 1순위: Railway

| 항목 | 내용 |
|------|------|
| **예상 비용** | **월 $5~15** (소규모 Streamlit 앱 기준, 약 **7,000~20,000원**) |
| **결제 구조** | Hobby $5/월 구독 + 사용량(CPU·RAM·트래픽). 소규모면 $5 크레딧으로 거의 커버 가능. 초과 시 $10/vCPU·월, $10/GB RAM·월 등 |
| **장점** | Streamlit 공식 가이드 있음, GitHub 연동·배포 간단, 상시 실행(잠자기 없음), 로그·재시작 편함 |
| **단점** | 사용량 늘면 비용 증가. 예산 알림 설정 권장 |
| **적합** | **상용화 전 테스트 + 초기 오픈**까지 한 곳에서 처리하고 싶을 때 |

- [Railway – Deploy Streamlit](https://railway.com/deploy/streamlit) 참고하여 배포 가능.

---

### 2순위: Google Cloud Run

| 항목 | 내용 |
|------|------|
| **예상 비용** | **월 $0~15** (트래픽 적으면 무료 구간 안에서 가능. 많으면 사용량만큼 과금, 약 **0~20,000원**) |
| **결제 구조** | 요청·vCPU·메모리·트래픽 종량제. 매월 **무료 200만 요청 + vCPU/메모리 무료 한도** 있음 |
| **장점** | 무료 한도 넉넉, 트래픽 적을 때 거의 무료. 이미 Gemini/Firebase 쓰면 같은 GCP 계정에서 관리 가능 |
| **단점** | Docker 이미지 빌드·설정 필요. Streamlit을 컨테이너로 패키징하는 작업 필요 |
| **적합** | **비용 최소화**하면서 상시 실행이 필요할 때, 또는 이미 GCP 사용 중일 때 |

| 항목 | 내용 |

| **단점** | 무료는 슬립 있음. 상시 실행하려면 유료 필수 |

| 항목 | 내용 |
|------|------|
| **예상 비용** | **월 약 $4~8** (소규모 앱, 약 **5,500~11,000원**) |

| **장점** | 글로벌 엣지, 배포 빠름 |

## 상용화 후 (트래픽 증가 시)

- **Railway**: 사용량에 따라 $20~50/월까지 늘어날 수 있음. 예산 알림 설정 필수.
- **Cloud Run**: 요청·동시 접속이 많아지면 비용이 구간별로 올라감. 예산·할당량 알림 설정 권장.
- **Render Production** 등 고정 플랜: 트래픽이 매우 많아지면 상위 플랜 또는 다른 플랫폼 검토.

## 상용화 염두에 둔 추천 (Production 추천)

유료 사용자·안정 서비스·스케일을 고려할 때의 우선순위입니다.

### 1순위: **Railway** (상용화 1순위)

| 기준 | 내용 |
|------|------|
| **안정성** | 상시 실행, 잠자기 없음. 재시작·배포가 단순함. |
| **비용** | 초기 **월 $5~15**, 트래픽 늘면 **$20~50** 수준까지 예상. 예산 알림으로 비용 제어 가능. |
| **운영** | GitHub 연동·자동 배포, 로그·메트릭 확인 용이. Streamlit 공식 가이드 있음. |
| **확장** | 리소스(CPU·RAM) 증설로 대응 가능. 나중에 트래픽 많이 늘면 별도 백엔드 분리 검토. |
| **적합** | **첫 상용 오픈 ~ 초기 성장**까지 한 플랫폼에서 운영하고 싶을 때. |

→ **상용화 염두면 Railway를 먼저 추천.** 이전도 쉽고, 비용·운영 균형이 좋음.

---

### 2순위: **Google Cloud Run**

| 기준 | 내용 |
|------|------|
| **안정성** | GCP 인프라, SLA·가용성 좋음. min instances ≥ 1 이면 상시 대기. |
| **비용** | 트래픽·동시 접속에 따라 종량제. 무료 한도 넉넉해 **초기엔 $0~10** 가능. 늘면 $20~50+ |
| **운영** | Docker 빌드·배포 파이프라인 필요. 이미 **Gemini·Firebase** 쓰면 GCP 한 곳에서 관리 가능. |
| **확장** | 트래픽에 맞춰 자동 스케일. 상용화 후 트래픽 많이 늘어도 대응 가능. |
| **적합** | **비용 최소화 + 장기 스케일**을 같이 고려할 때, 또는 이미 GCP 사용 중일 때. |

→ 상용화 후 트래픽이 커져도 확장성·신뢰도 면에서 유리.

---

### 3순위: **Render** (고정비 선호 시)

| 기준 | 내용 |
|------|------|
| **안정성** | 유료 플랜은 슬립 없음. 플랜별 리소스 고정. |
| **비용** | **Starter $7/월**, **Production $25/월** 등 **월 고정비**로 예산 예측 쉬움. |
| **운영** | GitHub 연동, `render.yaml`로 인프라 코드화 가능. |
| **적합** | **월 예산을 고정**하고 싶을 때. |

---

### 상용화 시 피할 것

- **Streamlit Community Cloud 무료**: 잠자기·리소스 제한으로 상용 서비스에는 부적합. 테스트·데모용으로만.
- **Fly.io**: 비용은 낮지만 상용화 시 SLA·지원·운영 체감은 Railway·GCP보다 약함. 소규모·비용 최우선일 때만.
